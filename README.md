# russia-catchall

러시아·벨라루스 수출 건의 제재품목, 사전신고 유형, 의심징후, 거래자 위험 신호를 순서대로 확인하고 통상그룹 품의 초안과 결과보고서를 만드는 내부 업무지원 도구입니다.

사용자가 제도나 판정 기준을 자세히 알지 않아도 화면의 질문에 답하면 다음 절차가 안내되도록 설계했습니다.

> 이 도구는 업무 진행을 돕는 내부 점검 도구입니다. 화면의 자동 분류만으로 수출 가능 여부가 확정되는 것은 아닙니다. 통상그룹 문의 또는 품의가 안내된 건은 반드시 해당 절차를 진행해야 합니다.

## 바로가기

- 배포 화면: <https://jinhsong.github.io/russia-catchall/>
- 일반 로컬 실행: `index.html`
- 관리자 테스트: `index.html?admin=1`
- 수동 테스트 대시보드: `test-environment.html`

## 주요 특징

- `index.html` 하나로 실행되는 정적 웹 도구
- 자가판정·전문판정·CSL 이름 자료를 gzip으로 압축하여 HTML 내부에 탑재
- 외부 JavaScript 라이브러리, 외부 API, 서버 DB, 로그인 없이 브라우저에서 처리
- 모델명 완전 일치 조회와 HS Code 접두 매칭
- 모델별 사전신고 유형 1개 선택
- 복수 인보이스와 인보이스별 복수 품목 입력
- 구매자·최종수하인·최종사용자 CSL 유사 일치 확인
- 군·정부·방산·항공우주·군사 용도·WMD 관련 키워드 확인
- 14개 의심징후 질문과 SERC/SRR 간소화 경로
- AI 검토결과에서 입력값과 DB 조회결과를 표로 통합 확인
- 바탕체 12포인트 품의 초안과 결과보고서 생성
- 결과보고서 `.html` 다운로드
- 일반 사용자 화면과 관리자 테스트 시나리오 화면 분리

## 현재 사용자 흐름

```text
1. 수출품목 입력
        │
        ├─ 검토대상·DB 미등록·판정 만료 포함
        │      └─ 사양·용도 입력 → 확인 질문 → 판정 요청 품의
        │
        └─ 모든 품목의 유효 판정 확인 + 제재대상 HS
               └─ 사전신고 유형 선택 → 거래정보 입력 → 확인 질문

2. 거래정보 입력
3. 확인 질문
4. AI 검토결과
5. 품의 초안 또는 통상그룹 문의
```

상위 단계의 모델, HS Code, 유형, 거래정보 또는 질문 답변을 수정하면 이후 단계의 결과, CSL 확인값, 품의 초안이 무효화됩니다. 다시 `다음`을 눌러 최신 입력값으로 검토해야 합니다.

품의 초안에서 사용자가 직접 고친 제목과 본문은 AI 검토결과로 돌아갔다가 다시 진입해도 유지되며, 결과보고서에도 같은 편집본이 들어갑니다. 단, 품목·거래정보·확인 질문 등 상위 입력을 수정하면 기존 초안은 폐기되고 최신 입력값으로 다시 생성됩니다.

## 1단계 — 수출품목 입력

### 입력 항목

- 모델명
- 제품명
- HS Code: 숫자 6~10자리
- 사양: 판정 요청 대상일 때 필수
- 용도: 판정 요청 대상일 때 필수

모델은 한 건씩 추가하거나 대량 입력할 수 있습니다. 대량 입력은 모델명만 한 줄에 하나씩 입력합니다.

### 모델 DB 조회

- 모델명은 대소문자와 공백을 제거한 뒤 완전 일치로 비교합니다.
- 부분 문자열은 자동 확정하지 않습니다.
- 같은 모델의 판정이 여러 개면 다음 순서로 선택합니다.
  1. 판정일·유효기간 형식이 정상인 판정
  2. 유효한 판정
  3. 전문판정
  4. 최근 판정일
- 판정 유효기간은 DB의 유효기간을 우선 사용하고, 없으면 판정일로부터 2년으로 계산합니다.
- 존재하지 않는 날짜나 판정일·유효기간이 모두 없는 DB 행은 자동으로 유효 판정 처리하지 않고 판정 요청 대상으로 보냅니다.
- DB에서 선택한 모델의 제품명, HS Code 등 DB 파생값은 사용자가 임의로 변경할 수 없습니다.
- 모델을 직접 변경하면 이전 DB 파생값도 함께 초기화됩니다.
- 테스트용 샘플과 실제 판정 DB에 같은 모델이 있으면 실제 판정 DB 값을 우선 사용합니다.

### DB에 없지만 실제 판정이 완료된 경우

`판정 DB에는 없지만 실제 판정 완료`를 선택하고 다음 값을 직접 입력할 수 있습니다.

- 판정발급번호: 숫자 정확히 10자리
- 판정 유효기간: 판정 발급일로부터 2년

수기 판정번호가 10자리가 아니거나 유효기간이 없거나 이미 만료된 경우에는 유효 판정으로 처리하지 않습니다. 수기 정보는 결과보고서에 수기 입력으로 표시됩니다.

### 제재품목 HS 판정

- HS Code에서 숫자만 사용합니다.
- 내장된 대러 제재 HS 목록의 코드로 시작하면 `제재대상`으로 분류합니다.
- 제재 HS와 일치하지 않으면 `검토대상`으로 분류합니다.

### 다음 단계 분기

입력 품목 중 하나라도 아래 조건에 해당하면 전체 거래는 판정 요청 경로로 진행합니다.

- 제재 HS 목록과 일치하지 않는 검토대상
- 판정 DB 미등록
- 판정 유효기간 만료

이 경우 사양과 용도를 추가로 받고 거래정보 입력을 건너뛴 뒤 확인 질문과 판정 요청 품의로 이동합니다.

모든 품목이 유효한 판정정보를 가지고 있고 제재대상 HS로 분류된 경우에만 사전신고 유형과 거래정보 입력으로 이동합니다.

같은 모델은 대소문자나 공백만 다르게 중복 입력할 수 없습니다.

## 2단계 — 거래정보 입력

### 품목별 사전신고 유형

각 품목에는 아래 유형 중 정확히 하나만 선택할 수 있습니다.

| 유형 | 추가 확인 또는 준비사항 |
|---|---|
| 소비자 통신제품 | 품목별 출시 여부 확인 |
| 의료기기 또는 전용 부품 | 설치확인서·이행점검보고서 등 추가 증빙 제출 가능 여부 확인 |
| 수입한 물품을 다시 반송 | 과거 수입 사실을 증명하는 자료 필요 |
| 전시회·박람회 출품 | 1년 이내 재수입 또는 현지 폐기 및 증빙 가능 여부 확인 |
| 수리·검사·시험 후 재반입 또는 폐기 | 1년 이내 재수입 또는 현지 폐기 및 증빙 가능 여부 확인 |
| 어느 항목에도 해당하지 않음 | 통상그룹 문의 경로로 이동 |

전시회·박람회와 수리·검사·시험 유형의 추가 질문은 내용이 같으므로 같은 품목에 중복 질문하지 않습니다.

거래 전체에 소비자 통신제품과 반송 유형이 함께 존재하면 상대적으로 적용이 쉬운 소비자 통신제품 유형 우선 검토를 권장합니다.

품목 중 하나라도 `어느 항목에도 해당하지 않음`이면 사전신고 품의를 만들지 않고 통상그룹 문의로 이동합니다. 이때 다음 값을 필수로 입력합니다.

- 구매자
- 최종수하인
- 최종사용자
- 최종사용용도
- 수출예정일

### 인보이스와 품목

- 인보이스를 여러 개 추가할 수 있습니다.
- 각 인보이스에 품목을 여러 개 추가할 수 있습니다.
- 같은 인보이스 안에 같은 모델을 중복 추가할 수 없습니다.
- 인보이스 번호는 대소문자와 공백을 무시하여 중복 검사합니다.

인보이스별 입력값은 다음과 같습니다.

- 인보이스 번호
- 목적국: 러시아 또는 벨라루스
- 구매자
- 최종수하인
- 최종사용자
- 최종사용용도

품목별 입력값은 다음과 같습니다.

- 모델
- 수량: 1 이상의 정수
- 단가(USD): 유상 거래는 0보다 큰 값
- 유무상: 유상 또는 무상

러시아를 선택하면 구매자·최종수하인·최종사용자를 `SERC`, `SRR`, `기타` 중에서 선택합니다. `기타`는 실제 거래자 이름을 직접 입력합니다. 벨라루스는 거래자 이름을 직접 입력합니다.

## 거래자 CSL 및 키워드 확인

### Consolidated Screening List

- 구매자, 최종수하인, 최종사용자의 이름을 로컬 CSL 자료와 비교합니다.
- Jaro–Winkler 점수 90점 이상을 유사 일치로 봅니다.
- SERC와 SRR은 CSL 비교에서 제외합니다.
- 비교 후보를 이름 길이별로 먼저 좁히고 일정 건수마다 화면에 제어권을 돌려주어, 다수 인보이스 확인 중에도 화면이 장시간 멈추지 않도록 처리합니다.
- CSL 자료 기준일은 코드의 `CSL_DATA_DATE`에서 관리합니다.
- 현재 내장 기준일은 `2026-07-12`입니다.

CSL 일치는 의심징후 13번을 자동으로 `예` 처리합니다. 품의에는 일치 여부를 `O`로 간단히 표시하고, 입력명·일치명·점수·출처 등 상세내용은 결과보고서에 기록합니다.

CSL은 로컬 스냅샷이므로 실시간 미국 정부 조회를 대체하지 않습니다.

### 최종사용자·최종사용용도 키워드

국문과 영문 키워드를 대소문자 구분 없이 확인합니다. 영문은 `defense/defence`, `armored/armoured` 등 주요 미국·영국식 철자 차이를 포함합니다.

확인 범주는 다음과 같습니다.

- 군 및 정부기관
- 방산 및 항공우주
- 군사적 전용 및 개조
- 핵·원자력·생화학무기 등 WMD 우려 분야

발견된 키워드는 AI 검토결과와 결과보고서에 표시됩니다. 품의 표에서는 관련 최종사용용도를 굵게 표시합니다.

## 3단계 — 확인 질문

14개 의심징후에 `예` 또는 `아니오`로 답합니다. `모두 아니오 선택` 버튼을 사용할 수 있습니다.

구매자·최종수하인·최종사용자가 모두 SERC 또는 SRR이면 다음과 같이 간소화됩니다.

- 11번 재수출 관련 질문만 직접 입력
- 나머지 항목은 자동으로 `아니오`
- CSL 자동 확인 결과는 13번에 반영

일반 거래에서 사용자가 직접 선택한 의심징후가 하나라도 `예`이면 품의 생성을 차단하고 결과보고서를 첨부하여 통상그룹에 문의하도록 안내합니다.

세 거래자가 모두 SERC/SRR인 경로는 수동 의심징후가 있어도 품의까지 진행할 수 있으며, 발견된 의심징후를 품의에 표시합니다.

## 4단계 — AI 검토결과

사용자가 다음에 해야 할 일을 가장 먼저 표시합니다.

- 판정 요청 품의
- 사전신고 품의
- 통상그룹 문의
- 결과 저장

품의가 필요한 경우 다음 순서로 안내합니다.

1. 다음 단계의 품의 초안에서 제목과 본문 복사
2. 통상그룹 합의
3. 파트장 결재로 상신
4. 결과보고서 `.html` 첨부

화면에는 아래 내용이 접히지 않은 표로 표시됩니다.

- 수출품목과 판정 DB 조회결과
- 제재품목 분류, 판정결과, 판정발급번호, 유효기간
- 인보이스·거래자·품목·수량·단가·유무상
- 14개 확인 질문 답변
- 발견 키워드
- CSL 일치 결과

`결과보고서(.html) 다운로드` 버튼 아래의 `(품의에 첨부 필요)` 표시는 실제 품의가 생성되는 경로에서만 나타납니다.

## 5단계 — 품의 초안 또는 통상그룹 문의

### 판정 요청 품의

제목 형식은 다음과 같습니다.

```text
상황허가 대상품목 판정 요청 (모델명)
상황허가 대상품목 판정 요청 (모델명 등 n개 품목)
```

한 품목일 때는 `등`, `각 제품별`처럼 부자연스러운 복수 표현을 사용하지 않습니다.

본문 표에는 HS Code, 품목명, 모델명, 사양, 용도, 비고를 표시합니다. 만료된 판정은 비고에 만료 사유와 만료 판정번호를 함께 표시합니다.

기본 첨부는 다음과 같습니다.

```text
#1. 결과보고서(.html) 1부
#2. 사양서 또는 카탈로그 1부
```

복수 품목이면 두 번째 첨부를 품목별 자료로 안내합니다.

### 사전신고 품의

제목에는 인보이스 번호를 쉼표로 구분하여 최대 5개까지 표시합니다. 5개를 초과하면 나머지는 `외 n건`으로 표시합니다.

```text
[국가]향 사전신고 요청 (INV-001)
[국가]향 사전신고 요청 (INV-001, INV-002, INV-003)
[국가]향 사전신고 요청 (INV-001, INV-002, INV-003, INV-004, INV-005 외 2건)
```

본문의 거래·품목정보 표는 인보이스 기준으로 번호를 부여합니다.

- 목적국 러시아: `RU`
- 목적국 벨라루스: `BY`
- 표의 품목 표시 순서: 품목명 → 모델명 → 판정발급번호
- HS Code는 사전신고 품의 표에 표시하지 않음
- CSL 관련 거래자 또는 키워드 관련 최종사용용도는 굵게 표시
- 굵게 표시된 항목이 있으면 표 아래에 사유 표시
- CSL 상세 점수와 목록은 결과보고서에서 확인

소비자 통신제품은 사전신고 유형을 다음과 같이 표시합니다.

- `소비자 통신제품(출시)`
- `소비자 통신제품(미출시)`

의료기기·전시회·수리 유형의 추가 증빙 답변은 별도 열을 만들지 않고 유형명 아래에 작게 표시합니다.

### 유형별 첨부

모든 품의 첨부는 `#1`, `#2`, `#3` 순서로 자동 번호를 부여합니다.

| 조건 | 추가 첨부 문구 |
|---|---|
| 공통 | 결과보고서(.html) 1부 |
| 소비자 통신제품·미출시 | 6개월 이내 출시 확약서, 확약서 결재 증빙(.eml) |
| 소비자 통신제품·출시 | 삼성닷컴 등 오픈마켓 판매 증빙 |
| 의료기기 | 설치확인서·이행점검보고서 등 최종사용자·사용용도 입증자료 |
| 수입물품 반송 | 과거 수입 사실 증빙(수입신고필증 등) |
| 전시회·박람회 | 전시회 또는 박람회 출품 증빙 |
| 수리·검사·시험 | 수리·검사·시험 증빙 |

인보이스 및 거래관계 증빙과 모델별 판정자료는 기본 첨부목록에 자동 추가하지 않습니다.

- 제목과 본문은 화면에서 직접 수정 가능
- 제목 복사, 본문·표 복사 버튼 제공
- 결과보고서도 바탕체 12포인트로 생성

## 결과보고서

AI 검토결과와 품의 초안 화면에서 `결과보고서(.html) 다운로드`를 누르면 독립 실행형 보고서를 생성합니다.

보고서에는 다음 내용이 포함됩니다.

1. 수출품목과 판정정보
2. 거래정보와 품목별 사전신고 유형
3. 확인 질문, 키워드, CSL 상세결과
4. AI 검토결과와 진행 안내
5. 판정 요청 또는 사전신고 품의 초안

보고서는 브라우저에서 열거나 메일·품의에 첨부할 수 있습니다. CSL 입력명·일치명·점수·출처 등 화면보다 상세한 확인정보도 보고서에 남습니다.

## 실행 방법

### 가장 간단한 방법

최신 Edge 또는 Chrome에서 `index.html`을 엽니다. 모든 필수 자료가 HTML에 내장되어 있어 인터넷 연결이나 별도 데이터 파일이 필요하지 않습니다.

내장 DB 압축 해제에 브라우저의 `DecompressionStream` 기능을 사용합니다. 구형 브라우저에서 자료 로드 오류가 발생하면 최신 Edge 또는 Chrome을 사용하세요.

### 로컬 테스트 서버

Windows에서 `start-test-server.bat`를 더블클릭합니다.

기본 주소는 다음과 같습니다.

```text
http://127.0.0.1:8765/test-environment.html
```

8765 포트가 사용 중이면 8766부터 사용 가능한 포트를 자동 선택하며, 실제 주소는 서버 창의 `Test URL`에 표시됩니다.

직접 포트를 지정하려면 다음과 같이 실행합니다.

```powershell
.\start-test-server.ps1 -Port 9000
```

브라우저를 자동으로 열지 않으려면 다음과 같이 실행합니다.

```powershell
.\start-test-server.ps1 -Port 9000 -NoBrowser
```

종료는 서버 창에서 `Ctrl+C`를 누릅니다.

## 관리자 테스트 시나리오

일반 주소에서는 테스트 시나리오가 표시되지 않습니다.

```text
index.html?admin=1
```

관리자 URL에서는 1단계에 39개 테스트 시나리오가 표시됩니다. 품목뿐 아니라 유형, 인보이스, 거래자, 수량, 단가, 유무상, 질문 답변까지 이후 단계에 자동 입력됩니다.

시나리오 범위는 다음을 포함합니다.

- 유효 판정, 판정 만료, DB 미등록, 수기 판정
- 제재대상과 검토대상 혼합
- 복수 인보이스와 복수 품목
- 품목별 서로 다른 사전신고 유형
- 출시·미출시 소비자 통신제품
- 의료기기·전시회·수리 추가질문 예/아니오
- SERC/SRR 자동 확인
- 키워드 및 CSL 일치
- 의심징후 예/아니오
- 중복 모델·인보이스·품목과 숫자 입력 오류

`?admin=1`은 화면 노출을 구분하는 기능일 뿐 인증이나 접근통제가 아닙니다. 테스트 데이터는 실제 판정이나 거래에 사용하면 안 됩니다.

수동 회귀검사는 `test-environment.html`의 체크리스트를 사용합니다.

## DB 갱신 시 주의사항

### 판정 DB

런타임 원본은 `index.html`의 `EMBEDDED_DB_GZIP.self`과 `EMBEDDED_DB_GZIP.expert`입니다.

CSV 원본의 주요 헤더는 다음과 같습니다.

| 필드 | 인식 가능한 대표 헤더 |
|---|---|
| 모델명 | `모델명(설명)`, `모델명`, `model`, `Model` |
| 제품명 | `제품명`, `품목명`, `물품명`, `item` |
| 판정결과 | `결과`, `판정결과`, `result` |
| 판정발급번호 | `판정번호`, `number` |
| 판정일 | `판정일`, `detDate` |
| 유효기간 | `유효기한`, `유효기간`, `expiry` |
| HS Code | `HSKCD`, `HS코드`, `HS`, `hs`, `HSK` |
| 사양 | `규격`, `사양규격`, `사양 및 규격`, `spec` |

### CSL DB

런타임 원본은 `EMBEDDED_DB_GZIP.csl`입니다. CSL을 교체할 때는 반드시 `CSL_DATA_DATE`도 실제 자료 기준일로 변경합니다.

### 제재 HS DB

런타임 원본은 `SANCTION_HS_PREFIXES`입니다. 비교 전에 HS Code에서 숫자가 아닌 문자를 제거하고, 긴 접두 코드부터 일치 여부를 확인합니다.

### 갱신 절차

1. 원본 CSV를 UTF-8로 정리합니다.
2. 중복, 빈 모델명, 잘못된 HS Code, 잘못된 날짜를 확인합니다.
3. CSV를 gzip으로 압축한 뒤 Base64 문자열로 변환합니다.
4. `index.html`의 해당 내장 상수를 교체합니다.
5. CSL 갱신이면 `CSL_DATA_DATE`를 변경합니다.
6. 일반 주소에서 관리자 시나리오가 숨겨지는지 확인합니다.
7. `index.html?admin=1`의 시나리오와 `test-environment.html` 체크리스트를 수행합니다.
8. 결과보고서와 품의 초안의 표·첨부·서식을 확인합니다.

현재 별도의 자동 임베드 빌드 스크립트는 저장소에 포함되어 있지 않습니다. DB를 갱신할 때는 원본 CSV만 교체하지 말고 내장 payload까지 함께 갱신해야 합니다.

## 개인정보·보안

- 일반 사용 중 입력값은 외부 API로 전송하지 않습니다.
- 판정 DB와 CSL은 브라우저 메모리에서 처리됩니다.
- 결과보고서는 사용자의 브라우저에서 생성됩니다.
- 외부 스크립트와 외부 XLSX 파서를 사용하지 않습니다.
- 결과보고서에는 거래자·최종사용자·용도·CSL 결과 등 민감한 업무정보가 포함될 수 있으므로 사내 보안정책에 따라 저장·전달해야 합니다.
- 로컬 CSL은 최신 제재목록을 자동 보장하지 않으므로 DB 기준일과 갱신 절차를 관리해야 합니다.

## 문제 해결

### 자료를 불러오지 못했다는 메시지가 표시됨

- 최신 Edge 또는 Chrome인지 확인합니다.
- 브라우저에서 `DecompressionStream`을 지원하는지 확인합니다.
- `index.html`이 임베드 과정에서 잘리지 않았는지 파일 크기와 Git diff를 확인합니다.
- 판정 DB 또는 CSL payload가 비어 있거나 손상되지 않았는지 확인합니다.
- 필수 DB 오류는 정상적인 미매칭으로 처리하지 않으며 다음 단계 진행을 차단합니다.

### 로컬 서버가 열리지 않음

- 서버 창의 `Test URL`이 실제 브라우저 주소와 같은지 확인합니다.
- 8765가 사용 중이면 스크립트가 표시한 다른 포트로 접속합니다.
- 방화벽 또는 보안 프로그램이 `127.0.0.1` 연결을 막는지 확인합니다.
- 서버 창을 닫으면 웹 페이지 연결도 종료됩니다.

### 한글이 깨짐

- 저장소 파일을 UTF-8로 유지합니다.
- 배치 파일은 UTF-8 코드페이지로 전환한 뒤 PowerShell 스크립트를 실행합니다.

### 이전 결과가 남아 보임

- 상위 입력을 변경하면 이후 단계가 자동 잠깁니다.
- 최신 입력으로 각 단계의 `다음`을 다시 눌러야 합니다.
- 배포 화면이 오래된 경우 강력 새로고침하거나 쿼리 문자열을 붙여 캐시를 우회합니다.

## 유의사항

이 도구는 내부 업무지원용입니다. 제재품목, 판정, 사전신고, 의심징후 및 CSL 자동 확인은 내장된 자료와 규칙을 기준으로 합니다. 법령·고시·제재목록·사내 절차가 바뀌면 코드와 DB를 함께 갱신해야 하며, 최종 업무 판단은 통상그룹 검토와 승인 절차를 따릅니다.
