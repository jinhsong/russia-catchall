# russia-catchall — 수출 판정·사전신고 점검 도구

모델명을 입력하면 판정정보를 확인하고, 필요한 판정신청·재판정·사전신고 양식을 자동으로 채워주는 정적 웹 도구입니다.
사용자는 마지막에 **결과 보고서(.html)** 를 내려받아 담당자에게 제출합니다.

## 배포 (GitHub Pages)

1. 이 저장소를 GitHub에 푸시합니다.
2. **Settings → Pages → Build and deployment → Source: `Deploy from a branch`** 선택.
3. Branch를 `main`(또는 배포 브랜치) / 폴더 `/ (root)` 로 설정 후 저장.
4. 잠시 뒤 `https://<사용자>.github.io/russia-catchall/` 에서 접속됩니다.

> 로컬에서 더블클릭(`file://`)으로 열면 브라우저 보안정책상 DB 자동 로드가 막힐 수 있습니다.
> 그땐 화면의 **"파일 직접 선택"** 으로 CSV를 불러오거나, 간단한 로컬 서버(`python -m http.server`)로 여세요.

## DB 갱신 방법 (정기 제공)

`data/` 폴더의 CSV 파일 내용만 교체해서 푸시하면 됩니다. 엑셀에서 **CSV(UTF-8)** 로 저장하세요.
헤더(첫 줄) 이름은 그대로 유지해야 합니다.

| 파일 | 용도 | 헤더(컬럼) |
|---|---|---|
| `data/self_classification.csv` | 자가판정결과 | `모델명,품목명,판정결과,판정번호,유효기한` |
| `data/expert_classification.csv` | 전문판정결과 | `모델명,품목명,판정결과,판정번호,유효기한` |
| `data/keyword_hs.csv` | 키워드→HS 추천 | `키워드,HS코드,품목설명` |
| `data/russia_sanctions_hs.csv` | 대러 제재대상 품목 | `HS코드,품목설명` |
| `data/prenotification_hs.csv` | 사전신고 대상 | `HS코드,품목설명` |

- `판정결과` 는 `해당` / `비해당` 으로 기재.
- `유효기한` 은 `YYYY-MM-DD` 형식. 오늘보다 과거면 자동으로 "만료 · 재판정 필요" 처리됩니다.
- 같은 모델이 자가/전문에 모두 있으면 **전문판정**을 우선 표시합니다.
- HS 매칭은 자리수가 달라도 됩니다(앞자리 접두 매칭). 예: 제재목록 `8542` ↔ 입력 `8542.31-0000` 매칭.

## 양식 교체 방법

`forms/` 폴더의 텍스트 파일 내용을 실제 양식으로 교체하세요.
입력값은 양식 안에 **`{변수}`** 형태로 넣으면 자동 치환됩니다. `※` 로 시작하는 안내 줄은 출력 시 제거됩니다.

| 파일 | 양식 | 사용 가능한 `{변수}` |
|---|---|---|
| `forms/reclassification.txt` | 재판정 요청서 | `{판정번호}`(만료번호) `{모델명}` `{품목명}` `{판정종류}` `{판정결과}` `{유효기한}` `{오늘}` |
| `forms/classification_request.txt` | 판정 신청서 | `{HS코드}` `{추천HS}` `{추천사용여부}` `{물품명}` `{모델명}` `{사양및규격}` `{무게}` `{용도}` `{사양서첨부}` `{오늘}` |
| `forms/prenotification.txt` | 사전신고서 | `{구매자}` `{최종수하인}` `{최종사용자}` `{모델목록}` `{의심징후체크결과}` `{오늘}` |

## 사용자 흐름

1. **모델명 입력** — 쉼표/공백/탭/줄바꿈으로 여러 개 입력.
2. **조회 결과** — 모델별 자가/전문, 판정결과(해당/비해당), 판정번호, 유효기한 표시.
   - 만료 시 **재판정 요청서**(만료 판정번호 포함) 자동 생성 + 복사 버튼.
3. **판정 신청** — 판정 이력이 없는 모델은 HS코드·물품명·사양/무게(필수)·용도 입력. 키워드로 HS 추천, 추천 미사용 시 표시. 사양서 첨부 안내.
4. **위험 점검** — 추천 HS 매칭이 없거나 제재/사전신고 HS에 해당하면 안내. 대러 제재 가능성 표시, 최종사용자 입력(품목 일괄 적용 선택), **상황허가 의심징후 12가지** 체크(수출 1회 1번).
5. **사전신고** — 사전신고 대상 HS에 해당하는 경우 구매자·최종수하인·최종사용자 및 모델별 판정번호(자동)·수량·단가(USD)·용도 입력 → 사전신고서 생성.
6. **결과 보고서 다운로드** — 모든 내용과 채워진 양식이 담긴 `.html` 보고서를 내려받아 제출.

> 본 도구는 점검 보조용이며 최종 책임은 수출 담당 부서의 검토에 따릅니다.
