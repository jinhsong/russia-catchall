# russia-catchall

러시아·벨라루스 수출 건의 제재품목, 사전신고 유형, 의심징후와 거래자 위험 신호를 순서대로 확인하고 품의 초안과 결과보고서를 만드는 내부 업무지원 도구입니다.

## 실행

저장소에는 실행 파일인 `index.html`과 이 안내문만 있습니다.

- 일반 화면: `index.html`
- 관리자 테스트 시나리오: `index.html?admin=1`
- 배포 화면: <https://jinhsong.github.io/russia-catchall/>

최신 Chrome 또는 Edge에서 `index.html`을 직접 열면 됩니다. 별도 설치, 서버, 인터넷 연결이 필요하지 않습니다.

## 단일 HTML 구성

`index.html` 안에 다음 요소가 모두 포함되어 있습니다.

- 화면 디자인과 전체 업무 흐름
- 자가판정·전문판정 샘플 DB
- 대러 제재품목 HS 접두 목록
- Consolidated Screening List 이름 자료
- 최종사용자·최종사용용도 확인 키워드
- 관리자 테스트 시나리오
- 결과보고서 `.html` 생성 기능

외부 JavaScript 라이브러리, 외부 API와 별도 데이터 파일을 사용하지 않습니다.

## 업무 흐름

1. 수출품목 입력
2. 거래정보 및 사전신고 유형 입력
3. 의심징후 확인
4. AI 검토결과 확인
5. 품의 초안 복사 또는 통상그룹 문의

상위 단계의 품목·거래정보·답변을 수정하면 이후 단계의 CSL 결과, AI 검토결과와 품의 초안이 무효화됩니다. 다시 `다음`을 누르면 최신 입력값으로 생성됩니다.

품의 초안에서 직접 수정한 제목과 본문은 상위 입력이 바뀌기 전까지 유지되며 결과보고서에도 반영됩니다.

## 관리자 테스트

주소 끝에 `?admin=1`을 붙이면 1단계에 관리자 테스트 시나리오가 표시됩니다.

```text
index.html?admin=1
```

테스트 시나리오는 유효·만료·미등록 판정, 복수 인보이스, 사전신고 유형, SERC/SRR, 키워드·CSL 일치, 의심징후와 입력 오류 경로를 포함합니다.

`?admin=1`은 화면 표시를 위한 옵션이며 인증 기능이 아닙니다. 테스트 데이터는 실제 업무에 사용하지 마세요.

## 데이터 갱신

실행 시 별도 CSV를 읽지 않습니다. 판정 DB나 CSL을 갱신할 때는 원본 자료를 정리한 후 gzip·Base64로 변환하여 `index.html`의 내장 데이터 상수를 교체해야 합니다.

주요 상수:

- `EMBEDDED_DB_GZIP.self`
- `EMBEDDED_DB_GZIP.expert`
- `EMBEDDED_DB_GZIP.csl`
- `SANCTION_HS_PREFIXES`
- `CSL_DATA_DATE`

데이터를 변경한 뒤에는 일반 화면과 관리자 테스트 시나리오에서 모델 조회, 단계 분기, CSL 확인, 품의 초안과 결과보고서를 함께 검증하세요.

## 주의

이 도구는 업무 진행을 돕는 내부 점검 도구입니다. 자동 분류만으로 수출 가능 여부가 확정되는 것은 아닙니다. 화면에서 통상그룹 문의 또는 품의가 안내된 건은 반드시 해당 절차를 진행해야 합니다.
