# russia-catchall — 상황허가 판정 · 사전신고 점검 도구

> 전략물자(품목 자체가 통제목록에 해당하는 경우)가 아니라, **상황허가 대상품목**(catch-all: 최종용도·최종사용자 등 상황 요건에 따라 허가가 필요한 품목)의 판정과 사전신고를 점검하는 도구입니다.

모델명을 입력하면 **상황허가 판정정보(자가/전문)** 를 확인하고, 상황에 따라 **재판정 요청서 · 판정 신청서 · 사전신고서** 를 자동으로 채워주는 **정적 웹 도구**입니다.
서버·데이터베이스·로그인 없이 브라우저에서만 동작하며(모든 처리는 사용자 PC에서 수행), 사용자는 마지막에 **결과 보고서(.html)** 를 내려받아 담당자에게 제출합니다.

- 입력 데이터는 외부로 전송되지 않습니다(정적 페이지).
- DB와 양식은 **저장소의 파일**이므로, 담당자가 파일만 교체·푸시하면 즉시 반영됩니다.

### 사용성 기능
- **진행 스테퍼**(상단): 현재 단계 표시 + 완료한 단계로 바로 이동. **한 번에 한 단계만 표시**(다음 단계로 가면 이전 단계는 숨김)되며, 스테퍼로 지나온 단계를 다시 열 수 있습니다.
- **임시저장**: 입력 내용을 브라우저에 자동 저장 → 새로고침해도 *‘이어서 작성’*으로 복원(저장소 전송 없음, 내 PC에만)
- **인라인 검증**: 누락/오류 필드를 빨간 테두리·메시지로 즉시 표시(품의·사전신고)
- **품의 제목/본문 직접 수정 가능 + ‘제목+본문 전체 복사’**
- **DB 미탑재 경고 배너**: 판정 DB가 로드되지 않으면 상단에 경고
- 금액 천단위 콤마, 추천 HS 칩 키보드 접근
- **모바일 대응**: 입력은 세로 스택, 결과 보고서의 넓은 표는 좁은 화면에서 카드형으로 전환
- **접근성**: 모든 입력 라벨을 `for`/`id`로 연결(스크린리더·클릭 영역 개선)

## 1. 전체 구조

```
russia-catchall/
├─ index.html                          # 사용자가 사용하는 단일 페이지 앱 (이 파일만 열면 동작)
├─ data/                               # DB (CSV) — 담당자가 정기적으로 교체
│   ├─ self_classification.csv         #   자가판정결과
│   ├─ expert_classification.csv       #   전문판정결과
│   ├─ keyword_hs.csv                  #   키워드 → HS코드 추천
│   ├─ russia_sanctions_hs.csv         #   대러 제재대상 품목(HS)
│   └─ prenotification_hs.csv          #   사전신고 대상(HS)
├─ forms/                              # 양식 템플릿 (txt) — 담당자가 실제 양식으로 교체
│   ├─ reclassification.txt            #   재판정 요청서
│   ├─ classification_request.txt      #   판정 신청서
│   └─ prenotification.txt             #   사전신고서
└─ README.md
```

- `index.html` 은 페이지가 열릴 때 `data/*.csv` 와 `forms/*.txt` 를 자동으로 읽어옵니다(`fetch`).
- 자동 로드에 실패한 환경(아래 FAQ 참고)에서는 화면의 **"파일 직접 선택"** 으로 CSV를 수동 로드할 수 있습니다.
- 외부 라이브러리·인터넷 연결이 필요 없습니다.

## 3. DB 갱신 방법

### 3.1 자가판정 / 전문판정 (xlsx)

자가판정·전문판정 결과는 **`(상황)자가판정…` / `(상황)전문판정…` 으로 시작하는 xlsx 파일**을 그대로 사용합니다.
두 파일을 각각 아래 이름으로 `data/` 폴더에 올려두면(덮어쓰기) 자동으로 읽습니다.

| 올릴 파일명(고정) | 원본 |
|---|---|
| `data/self.xlsx` | `(상황)자가판정…xlsx` |
| `data/expert.xlsx` | `(상황)전문판정…xlsx` |

- **제목행(헤더)은 2행**입니다. (1행은 비어 있거나 안내문이어도 됩니다.)
- 첫 번째 시트를 읽으며, 아래 컬럼을 가져옵니다(컬럼명이 정확히 일치해야 함).

  | 컬럼 | 용도 |
  |---|---|
  | `판정번호` | 판정번호 (재판정·사전신고서에 자동 기입) |
  | `HSKCD` | HS코드 (대러 제재·사전신고 대상 매칭에 사용) |
  | `제품명` | 제품명 표시 |
  | `모델명(설명)` | 모델명 조회 키 (설명이 함께 있어도 모델명이 포함되면 매칭) |
  | `규격` | 규격 표시 |
  | `판정일` | **유효기간 계산 기준** |
  | `결과` | `해당` / `비해당` |

- **유효기간 = 판정일 + 2년.** 오늘이 그 이후이면 자동으로 *"만료 · 재판정 필요"* 로 처리됩니다.
- 자가·전문 두 파일을 **통합**해 조회하며, 같은 모델에 판정이 여러 건이면 **유효 건 우선 → 전문판정 우선 → 최근 판정일** 순으로 한 건을 선택합니다.
- xlsx가 없을 때를 대비해 같은 이름의 데모용 CSV(`data/self_classification.csv`, `data/expert_classification.csv`)가 있으며, xlsx가 있으면 xlsx가 우선합니다. (CSV는 제목행이 1행, 컬럼은 xlsx와 동일)
- 사내망에서 xlsx 파서(CDN)가 차단되면 화면의 **"파일 직접 선택"** 으로 xlsx를 직접 올리거나 CSV로 대체하세요.

### 3.2 키워드-HS / 대러 제재 / 사전신고 (CSV)

`data/` 폴더의 CSV 파일 내용만 교체해서 푸시하면 즉시 반영됩니다. (엑셀에서 `CSV UTF-8` 로 저장, 첫 줄 헤더 유지)

| 파일 | 용도 | 헤더(첫 줄) |
|---|---|---|
| `data/keyword_hs.csv` | 키워드 → HS 추천 | `키워드,HS코드,품목설명` |
| `data/russia_sanctions_hs.csv` | 대러 제재대상 품목 | `HS코드,품목설명` |
| `data/prenotification_hs.csv` | 사전신고 대상 | `HS코드,품목설명` |
| `data/hs_codes.csv` | **유효 HS 10자리 목록**(판정 신청 HS 입력 허용 기준) | `HS코드,품목명` |

### 3.3 공통 규칙

- **판정결과**: `해당` / `비해당`.
- **모델명 매칭**: 대소문자·공백 무시, `모델명(설명)` 안에 입력 모델명이 포함되면 매칭.
- **키워드-HS**: `키워드` 가 입력 물품명/키워드에 포함되면 해당 HS를 추천. 짧고 일반적인 단어(예: `스마트폰`, `통신`, `센서`)로 등록할수록 매칭률이 높습니다.
- **HS 자리수**: 제재/사전신고/추천 목록의 HS는 6자리든 10자리든 무방합니다. **앞자리 접두 매칭**이라 목록에 `8517` 만 있어도 입력 `8517.13-0000` 과 매칭됩니다([6. HS코드 매칭 규칙](#6-hs코드-매칭-규칙) 참고).
- **`hs_codes.csv`(유효 HS DB)**: 판정 신청(품의)에서 HS는 이 목록에 있는 **10자리** 코드만 허용됩니다(없으면 입력 불가). 이 파일이 없으면 검증을 건너뛰고 경고만 표시합니다. ⚠ **앞자리 0이 있는 HS가 숫자로 저장되면 0이 사라질 수 있으니, HS 열은 반드시 텍스트로 저장**하세요(키워드-HS·제재·사전신고 DB도 동일).

## 4. 양식 교체 방법

`forms/` 폴더의 텍스트 파일 **내용을 실제 양식으로 교체**하세요.
양식 안에 **`{변수}`** 형태로 적어 두면 사용자가 입력한 값으로 자동 치환됩니다.

- `※` 로 시작하는 안내 줄은 출력(보고서·복사) 시 **자동 제거**됩니다.
- 치환할 값이 없는 변수는 `(미입력)` 으로 표시됩니다.
- 등록된 변수명만 치환됩니다(아래 표). 오타가 나면 `(미입력)` 으로 보일 수 있으니 변수명을 정확히 사용하세요.

| 파일 | 양식 | 사용 가능한 `{변수}` |
|---|---|---|
| `forms/reclassification.txt` | 재판정 요청서 | `{판정번호}`(만료번호) `{모델명}` `{품목명}` `{판정종류}` `{판정결과}` `{유효기한}` `{오늘}` |
| `forms/classification_request.txt` | 판정 신청서 | `{HS코드}` `{추천HS}` `{추천사용여부}` `{물품명}` `{모델명}` `{사양및규격}` `{무게}` `{용도}` `{사양서첨부}` `{오늘}` |
| `forms/prenotification.txt` | 사전신고서 | `{구매자}` `{최종수하인}` `{최종사용자}` `{모델목록}` `{의심징후체크결과}` `{오늘}` |

- **재판정 요청서** 는 만료된 모델마다 1건씩 생성되며, `{판정번호}` 에 **만료된 판정번호**가 들어갑니다.
- **판정 신청서** 는 판정 이력이 없는 모델마다 1건씩 생성됩니다.
- **사전신고서** 의 `{모델목록}` 은 모델별 `판정번호 / 수량 / 단가(USD) / 용도` 가, `{의심징후체크결과}` 는 12가지 의심징후 체크 결과가 자동으로 채워집니다.

## 5. 사용자 화면 흐름 (판정 로직)

사용자는 위에서 아래로 단계가 펼쳐지는 화면을 따라갑니다.

### STEP 1 — 모델명 입력
- 모델명 여러 개를 한 번에 입력 — **한 줄에 하나씩** 또는 **쉼표( , )** 로 구분(띄어쓰기는 구분자가 아님). 입력하면 인식된 모델 개수·목록이 실시간으로 표시됩니다.

### STEP 2 — 판정결과 확인 (모델별) · 결과에 따른 분기
- **유효 · 해당 + 사전신고 비대상 HS** → **허가예외 확인(STEP 4)** (개별수출허가/허가예외 적용 여부 확인).
- **유효 · 해당 + 사전신고 대상 HS**(`prenotification_hs.csv` 매칭) → **사전신고(STEP 5)**.
- **유효 · 비해당**: 사전신고 불필요 → **red flag 점검(상황허가 의심징후, STEP 6)** 대상.
- **만료**(판정일+2년 경과) → **재판정 요청서**(만료 판정번호 포함) 자동 생성 + 복사.
- **판정정보 없음** → **판정 신청 품의(STEP 3)**. (DB가 실시간이 아니므로, 이미 판정된 품목이면 ‘사전신고 진행하기’도 제공)
- 카드에는 모델명·제품명·HS코드·규격·판정구분·판정결과·판정번호·판정일·유효기한(판정일+2년)이 표시됩니다.

### STEP 3 — 판정 신청 (신규 판정 — 품의 작성)
- 판정 이력이 없는 모델은 **위험 점검으로 넘어가지 않고** 판정 신청 **품의(제목·본문)** 를 생성합니다.
- **거래자 정보**: 수입국(러시아/벨라루스 다중 선택), 수입자(러시아 선택 시 SERC·SRR 선택 + 기타 직접 입력).
- **품목 정보(모델별)**: 물품명(제품명, **2글자 이상**), HS코드(**숫자 10자리**, 점·하이픈 자동 제거, `hs_codes.csv` 의 **유효 HS만 허용**, 물품명 입력 시 추천·자동완성), 사양 및 규격, **무게(필수)**, 용도. (모델명은 STEP 1 입력, **3글자 이상**)
- "판정 신청 품의 생성" → **제목**과 **본문(품목 정보는 실제 표)** 을 복사. **한글/워드/메일에 붙여넣으면 표로 들어갑니다**(서식 미지원 텍스트박스에는 정렬된 텍스트 표). 본문은 직접 수정도 가능하며, 표 미리보기가 함께 표시됩니다.
- 품목 표 끝의 **비고** 칸에는 HS를 직접 입력한 경우 `HS 직접입력` 으로 표기됩니다.
- 안내: ① 상신 시 제목·본문 복사 사용 ② 품목별 사양서 첨부 필요 ③ 통상그룹(송진혁/강부현 프로) 합의 — 파트장 이상 결재.
- 다른 분류(비해당/해당) 모델이 함께 있으면 품의 생성 후 허가예외·사전신고·red flag 버튼으로 이어서 진행할 수 있습니다.

### STEP 4 — 허가예외 확인 (유효 **해당 · 사전신고 비대상 HS**)
- 상황허가 대상(해당)이나 사전신고 대상 HS가 아닌 품목을 모아, **개별수출허가 또는 허가예외** 적용 가능 여부를 확인하도록 안내합니다.

### STEP 5 — 사전신고 (유효 **해당 · 사전신고 대상 HS** · 인보이스 기반)
- 해당이면서 HS가 사전신고 대상인 모델을 대상으로 진행합니다. (판정 DB는 실시간이 아니므로, 이미 판정된 품목이라면 ‘판정정보 없음’이어도 ‘사전신고 진행하기’로 진행)
- **1개 인보이스 = 1개 수출거래**, 한 인보이스에 여러 품목을 넣을 수 있고 **인보이스도 여러 개** 추가할 수 있습니다.
- 인보이스별 입력: **인보이스 번호**, **목적국(러시아/벨라루스 택1)**, **최종사용자(SERC·SRR·기타 택1, 러시아일 때만 SERC/SRR 활성화)**, **최종사용용도**.
- 품목별 입력: **모델명**(앞서 입력한 모델 중 선택) → 판정 DB 참조로 **제품명·판정발급번호 자동입력**(같은 모델 판정이 여러 건이면 제품명으로 선택), **수량(EA) · 단가(USD)**, **총액(USD)=수량×단가 자동계산**, **유무상(유상/무상)**. DB에 없으면 제품명·판정발급번호 직접 입력.
- 입력 내용은 결과 보고서에 인보이스별 표로 정리됩니다.

### STEP 6 — red flag 점검 (유효 **비해당** 모델 · 수출 1회당 1번)
- 비해당 품목은 상황허가(catch-all) red flag 점검 대상입니다.
- **대러 제재대상 점검**: 각 모델의 HS가 `russia_sanctions_hs.csv` 에 매칭되면 *"대러 제재대상 품목 가능성이 높습니다."* 안내.
- **최종사용자 입력**: 상호·수입국·사업분야. **"모든 품목에 동일 적용"** 체크를 해제하면 모델별로 따로 입력할 수 있습니다.
- **상황허가 의심징후(red flag) 12가지** 체크 (품목별이 아니라 **수출 1회당 1번**).

### STEP 7 — 보고서 작성 (다운로드)
- 위 모든 입력 내용과 자동 채워진 양식이 담긴 **`.html` 보고서**를 내려받습니다.
- 파일명 예: `상황허가점검결과_2026-06-23.html`. 이 파일을 담당자에게 제출합니다.

## 6. HS코드 매칭 규칙

- 비교 전 HS코드에서 **숫자만 남깁니다**(점·하이픈·공백 제거). 예: `9031.80-9099` → `9031809099`.
- **앞자리(접두) 매칭**: 한쪽이 다른 쪽의 앞부분이면 매칭으로 봅니다(최소 4자리).
  - 예) 목록 `8542` ↔ 입력 `8542310000` → **매칭**
  - 예) 목록 `851762` ↔ 입력 `8517` → **매칭**(짧은 쪽이 접두)
  - 예) 입력 `8517` ↔ 목록 `8542` → 비매칭
- 따라서 제재·사전신고 목록은 보통 **6자리(소호) 단위**로 등록해도 10자리 입력과 잘 매칭됩니다. 더 정밀하게 제한하려면 자리수를 길게 등록하세요.

## 7. 결과 보고서

다운로드되는 `.html` 보고서에는 다음이 순서대로 포함됩니다.

1. 작성일
2. 모델별 판정결과 (모델명·제품명·HS코드·규격·판정일·유효기한, 만료 시 재판정 요청서 포함)
3. 판정 신청 품의 (거래자 정보 + 제목·본문, 품목 정보 표)
4. 허가예외 확인 (해당 · 사전신고 비대상 품목 목록)
5. 사전신고 (인보이스별 거래자·품목 표 + 합계)
6. red flag 점검 (대러 제재 가능성, 최종사용자, 상황허가 의심징후 12가지 체크표)

- 브라우저에서 그대로 열어 읽거나 **인쇄 → PDF로 저장** 할 수 있습니다.
- 스크립트가 포함되지 않은 정적 문서라 메일 첨부·보관에 적합합니다.

## 8. 자주 묻는 문제 (FAQ / 트러블슈팅)

**Q. "DB 미로드" 로 표시돼요.**
- GitHub Pages 등 웹 서버로 접속하면 자동 로드됩니다.
- 파일을 더블클릭해서 `file://` 로 열면 브라우저 보안정책(CORS)상 자동 로드가 막힙니다.
  → 화면의 **"DB를 불러오지 못했나요? 파일 직접 선택"** 에서 CSV를 직접 선택하거나,
  → 폴더에서 로컬 서버를 띄워 여세요: `python -m http.server` 후 `http://localhost:8000`.

**Q. 한글이 깨져요.**
- CSV를 **UTF-8** 로 저장하세요(엑셀: `CSV UTF-8(쉼표로 분리)(*.csv)`).

**Q. 모델이 조회되지 않아요.**
- 모델명 철자/하이픈을 DB와 동일하게 입력했는지 확인하세요(대소문자·앞뒤 공백은 무시됨).

**Q. 추천 HS가 안 떠요.**
- `keyword_hs.csv` 의 `키워드` 가 입력한 물품명/키워드에 포함돼야 합니다. 일반적인 키워드를 추가하세요.

**Q. 양식의 변수가 `(미입력)` 으로 나와요.**
- 양식의 변수명이 [4번 표](#4-양식-교체-방법)의 이름과 정확히 일치하는지 확인하세요(오타·다른 이름은 치환되지 않음).

## 9. 유의사항

본 도구는 사내 상황허가 판정·사전신고 **점검 보조용**입니다. 자동 안내(대러 제재 가능성, 사전신고 대상 여부 등)는 제공된 DB를 기준으로 한 **참고 정보**이며, **최종 판단과 책임은 수출 담당 부서의 검토**에 따릅니다. DB·양식의 정확성과 최신성 유지가 결과 신뢰도에 직접 영향을 줍니다.
