# data/ — 판정·제재 DB 파일 안내

> **현재 이 폴더의 CSV는 동작 확인용 샘플 데이터입니다.** 실제 DB 파일을 준비되는 대로 아래 규칙에 맞춰 교체(푸시)하면 도구에 즉시 반영됩니다. 코드 수정은 필요 없습니다.

## 로드 우선순위

| 구분 | 1순위 (xlsx) | 2순위 (csv, 대체용) |
|---|---|---|
| 자가판정결과 | `data/self.xlsx` (헤더 2행) | `data/self_classification.csv` |
| 전문판정결과 | `data/expert.xlsx` (헤더 2행) | `data/expert_classification.csv` |
| 키워드-HS 마스터 | — | `data/keyword_hs.csv` |
| 대러 제재대상 품목(HS) | — | `data/russia_sanctions_hs.csv` |
| 사전신고 대상(HS) | — | `data/prenotification_hs.csv` |

- `self.xlsx` / `expert.xlsx` 를 올리면 같은 이름의 CSV보다 **우선 사용**됩니다. xlsx가 없으면 CSV를 읽습니다.
- xlsx는 **첫 번째 시트**를 읽고, **컬럼 제목이 2행**에 있다고 가정합니다(실제 판정결과 xlsx 양식 그대로 사용).
- 파일이 둘 다 없거나 읽기에 실패하면 화면 상단에 "판정 DB가 로드되지 않았습니다" 경고가 표시되고, 화면 하단 **"판정 데이터 탑재 상태"** 패널에서 파일을 직접 선택해 임시 로드할 수 있습니다(브라우저에만 반영, 저장소 미변경).

## 컬럼 규칙

### 자가판정결과 / 전문판정결과

| 컬럼 | 필수 | 비고 |
|---|---|---|
| 판정번호 | O | |
| HSKCD (또는 HS코드) | O | 숫자 10자리 권장 |
| 제품명 | O | |
| 모델명(설명) | O | 조회 키. 부분일치로 검색됨 |
| 규격 | | |
| 판정일 | O | `YYYY-MM-DD` (`.` `/` 구분자도 허용). **유효기한 = 판정일 + 2년**으로 자동 계산 |
| 결과 | O | `해당` / `비해당` 포함 문자열 |

컬럼명은 위 표기 외에 일부 별칭(`모델명`, `HS코드`, `판정결과` 등)도 인식하지만, 가급적 표준 컬럼명을 유지하세요.

### keyword_hs.csv (키워드 → HS 추천)

`키워드,HS코드,품목설명` — 판정 신청 시 제품명 입력에 따라 HS 10자리를 추천하는 데 사용. **HS 직접입력도 이 파일들에 등록된 10자리만 허용**되므로, 실제 사용하는 HS는 반드시 등록돼 있어야 합니다.

### russia_sanctions_hs.csv / prenotification_hs.csv

`HS코드,품목설명` — HS는 4자리 이상(앞자리 기준 매칭). 예: `8542`는 `8542.31-…` 전체와 매칭됩니다.

## 교체 시 주의

- 인코딩은 **UTF-8(BOM 허용)** 로 저장하세요. 엑셀에서 CSV 저장 시 "CSV UTF-8" 형식을 선택.
- 쉼표가 들어간 값은 따옴표(`"`)로 감싸면 됩니다(파서가 처리).
- 교체 후 GitHub Pages 반영까지 1~2분 걸릴 수 있습니다. 화면 하단 "판정 데이터 탑재 상태"에서 건수를 확인하세요.
