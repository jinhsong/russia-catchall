환경 상태

HTML과 필수 CSV가 모두 응답해야 정상적인 판정 테스트가 가능합니다.

기능 점검 체크리스트

각 테스트는 앱을 새로고침한 뒤 시작하면 결과를 비교하기 쉽습니다.

실제 업무정보는 입력하지 말고 아래 샘플 값만 사용하세요. 체크 상태는 이 브라우저에만 저장됩니다.

1. 샘플 모델 자동입력

드롭다운·DB 자동 연동
  1. 수출품목 입력에서 SM-S928N을 입력하거나 목록에서 선택합니다.
  2. 제품명과 HS Code를 수정해 봅니다.
  3. ‘자동 확인 내용’을 펼칩니다.
예상: 제품명과 HS Code가 자동 입력되고 수정할 수 없음. 화면에는 ‘품목 확인이 완료되었습니다’가 표시되고 상세 정보는 접힌 영역에만 표시됨

2. 모델 변경 시 자동입력 초기화

이전 모델 데이터 잔존 방지
  1. SM-S928N 선택 후 모델명을 UNKNOWN-MODEL로 바꿉니다.
  2. 제품명과 HS Code를 확인합니다.
예상: 이전 제품명·HS·자동 확인 내용이 모두 사라지고 직접 입력할 수 있음

3. 모델명 완전 일치 조회

부분 문자열 오판정 방지
  1. 모델명에 SM 또는 SM-S928N-X를 입력합니다.
  2. 기존 SM-S928N 판정이 자동 선택되지 않는지 확인합니다.
예상: 기존 모델 정보가 자동 입력되지 않음. 정확히 일치한 모델만 자료와 연결됨

4. 추가 확인 품목 경로

불필요한 거래정보 입력 생략
  1. UNKNOWN-MODEL, 제품명 테스트 장비, HS 9999999999를 입력합니다.
  2. 나타난 사양과 용도를 입력하고 다음을 누릅니다.
  3. 모든 확인 질문에 답한 뒤 결과와 문서를 확인합니다.
예상: 거래정보 입력을 건너뛰어 확인 질문으로 이동. AI 검토결과에는 통상그룹으로 판정 요청 품의를 진행하라는 안내가, 품의 초안에는 판정 요청 내용이 표시됨. 만료 품목은 비고에 ‘판정 유효기간 만료’와 만료된 판정번호가 함께 표시됨

5. 거래정보와 복수 인보이스

자동 확인 품목의 정상 경로
  1. SM-S928N을 선택하고 다음을 누릅니다.
  2. 거래 유형을 선택한 뒤 거래정보 화면으로 이동합니다.
  3. 목적국을 러시아로 선택합니다.
  4. 구매자·최종수하인·최종사용자에서 SERC, SRR, 기타를 각각 확인하고 기타 이름을 입력합니다.
  5. 목적국을 벨라루스로 바꿉니다.
예상: 유형 선택 단계에서 앞서 입력한 각 품목에 적용 유형을 정확히 1개만 선택할 수 있음. 거래 전체에 소비자 통신제품과 반송 후보가 함께 있으면 소비자 통신제품 우선 검토를 권장하며, 인보이스 단계에는 품목별 최종 선택 결과만 요약됨

6. ‘해당 없음’ 경로

불필요한 질문 생략
  1. SM-S928N으로 거래정보 입력까지 이동합니다.
  2. ‘어느 항목에도 해당하지 않음’을 선택합니다.
  3. 구매자·최종수하인·최종사용자·최종사용용도·수출예정일을 입력하고 다음을 누릅니다.
예상: 5개 문의정보가 모두 필수로 검증되고 확인 질문을 생략함. AI 검토결과와 결과보고서에 입력정보를 기재하여 HTML 파일과 함께 통상그룹에 문의하라는 안내가 표시됨

7. 누락·숫자 입력 오류

팝업 대신 화면 안에서 안내
  1. DB 미등록 품목에서 ‘실제 판정 완료’를 선택하고 판정발급번호에 문자 또는 9자리 숫자를 입력해 봅니다.
  2. 거래정보에서 필수 항목을 비우고 다음을 누릅니다.
  3. 수량 0, -1, 1.5와 유상 단가 0을 각각 입력해 봅니다.
예상: 판정발급번호는 숫자만 입력되고 정확히 10자리여야 진행됨. 거래정보의 누락·숫자 오류도 다음 화면으로 이동하지 않고 화면 안에 목록으로 표시됨

8. 이전 단계 수정과 결과 파일

오래된 결과 재사용 방지
  1. AI 검토결과에서 수출품목·DB 조회결과, 거래정보, 확인 질문, 키워드, CSL 결과가 모두 펼쳐져 있는지 확인합니다.
  2. 결과보고서를 내려받고 품의 초안으로 이동해 같은 다운로드 버튼이 있는지 확인합니다.
  3. 상단 수출품목 입력으로 돌아가 모델 또는 HS를 바꿉니다.
예상: AI 검토결과에 사용자가 해야 할 일과 입력·조회 결과가 표로 한눈에 표시됨. 두 화면의 버튼은 ‘결과보고서(.html) 다운로드’이고, 품의가 필요한 건은 아래에 ‘(품의에 첨부 필요)’가 표시됨. 상위 입력을 바꾸면 뒤 단계가 잠김

9. SERC/SRR 확인 질문 자동입력

11번 재수출 질문만 수동 확인
  1. 러시아 인보이스의 구매자·최종수하인·최종사용자를 모두 SERC 또는 SRR로 선택합니다.
  2. 거래정보를 완료하고 확인 질문으로 이동합니다.
예상: SERC와 SRR은 CSL 비교에서 제외됨. 1~10번과 12~14번은 ‘아니오’로 자동 선택되고 수정할 수 없으며, 11번만 사용자가 예 또는 아니오를 선택함. 11번을 ‘예’로 선택해도 세 거래자가 모두 SERC/SRR이면 품의가 생성되고 해당 의심징후가 품의에 표시됨

10. 수동 ‘예’와 키워드 경로 구분

이상 항목만 의심징후에 표시
  1. 확인 질문에서 ‘모두 아니오 선택’을 눌러 전체가 선택되는지 확인합니다.
  2. 질문 하나를 ‘예’로 바꾸고 결과를 확인합니다.
  3. 별도 테스트에서 최종사용자에 Ministry of Defence, 최종사용용도에 armoured missile guidance를 입력하고 질문은 모두 아니오로 완료합니다.
예상: 사용자가 직접 ‘예’를 선택하면 통상그룹 문의 대상으로 처리됨. 키워드만 발견되면 품의가 생성되고 의심징후 검토에 발견 건수만 표시되며, 상세 키워드는 결과보고서에서 확인함. 이상 항목이 없으면 ‘이상 없음’으로 표시됨

11. CSL Jaro–Winkler 유사 일치

구매자·최종수하인·최종사용자 자동 스크리닝
  1. 러시아 거래자의 ‘기타’를 선택하고 한 항목에 China Telecom Corp Lmt를 입력합니다.
  2. 나머지 거래정보를 채우고 다음을 누릅니다.
  3. 확인 질문 13번과 최종 결과보고서를 확인합니다.
예상: CSL의 CHINA TELECOM CORP LTD와 90점 이상으로 일치하고 13번이 ‘예’로 자동 표시되지만 품의는 계속 생성됨. 품의 의심징후에는 CSL 일치 O만 표시되고, 입력명·등재명·점수·출처 목록은 결과보고서에만 기록됨

12. 품의 정보 통합 표

품목정보와 거래정보를 한 표로 확인·복사
  1. 품의가 허용되는 테스트를 완료하고 품의 초안으로 이동합니다.
  2. 판정 요청 또는 사전신고 요청 본문의 표를 확인합니다.
  3. ‘본문·표 복사’를 눌러 Word나 메일에 붙여넣습니다.
예상: 문서는 바탕체 12포인트이며 사전신고 표 No.는 인보이스 기준임. 소비자 통신제품은 ‘소비자 통신제품(출시)’ 또는 ‘소비자 통신제품(미출시)’로 표시되고, 다른 추가 증빙 답변은 별도 열 없이 유형 아래에 표시됨. 첨부는 결과보고서와 선택 유형에 필요한 서류만 번호가 부여됨

13. 이전 수정·중복 입력 회귀 테스트

오래된 결과와 중복 품목 차단
  1. 수출품목에 같은 모델을 대소문자만 다르게 2번 입력하고 다음을 누릅니다.
  2. 거래정보에서 같은 인보이스 번호를 2번 사용하고, 한 인보이스에 같은 모델을 2번 선택해 봅니다.
  3. AI 검토결과까지 진행한 뒤 이전 단계로 돌아가 모델·거래정보·질문을 각각 수정합니다.
  4. 품의 초안 제목과 본문을 수정하고 AI 검토결과로 돌아갔다가 다시 품의 초안으로 이동합니다. 결과보고서도 내려받아 수정 문구를 확인합니다.
  5. 그 상태에서 거래정보 하나를 수정하고 다시 진행해 품의 초안이 최신 입력값으로 재생성되는지 확인합니다.
  6. CSL 확인 중에는 이전 단계 버튼을 눌러 검색 완료 후 화면이 임의로 넘어가지 않는지 확인합니다.
예상: 중복 모델·중복 인보이스 번호·동일 인보이스 내 중복 품목은 다음 단계로 진행되지 않음. 품의 초안만 수정한 경우 뒤로 갔다 와도 편집 내용과 결과보고서가 유지됨. 상위 입력을 수정하면 뒤 단계가 다시 잠기고 초안이 최신 값으로 재생성되며, 오래된 CSL 결과가 덮어쓰지 않음

14. 시뮬레이션 시나리오

유효·만료·미등록·비제재·복수이력 분기
  1. index.html?admin=1 관리자 주소로 앱을 엽니다.
  2. 수출품목 입력의 ‘관리자 테스트 시나리오’를 펼칩니다.
  3. 각 시나리오 버튼을 누르고 단계별 ‘다음’만 눌러 진행합니다.
  4. 거래정보·수량·단가·유무상·확인 질문이 자동 입력되었는지 확인합니다.
  5. ‘복수 판정이력’에서는 자동 확인 내용을 펼쳐 판정번호를 확인합니다.
  6. ‘복합 경로’에서는 거래정보를 건너뛰는지 확인합니다.
예상: 39개 시나리오가 품목부터 이후 단계까지 자동 입력됨. 한 인보이스 안의 품목별 서로 다른 단일 유형, 복수 유형 후보 중 품목당 1개 선택, 해당 없음 혼합, 출시 전 소비자 통신제품, 의료·전시·수리 추가질문 아니오와 기존 판정·거래·오류 경로를 재현할 수 있음
